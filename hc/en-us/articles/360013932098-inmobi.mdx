This article is for publishers who want to use the DT FairBid SDK to load and display ads from InMobi via DT FairBid Mediation.

InMobi currently supports the following [ad types](https://developer.digitalturbine.com/hc/en-us/articles/360010915358-Ad-Types-and-Placement-ID):

* Banner
* Interstitial
* Rewarded

To integrate InMobi as a DT mediated network:

* [Step 1: Retrieve your InMobi parameters](#h_01J9W9C31TWB39TK3MG01T6EXE).
* [Step 2: Add InMobi as a DT mediated network](#h_01HN007B7WTB7QNA9P66MA0P94).
* [Step 3: Add the InMobi SDK to your integration](#h_01HMBMW7WQTXTTNNMYFPBHBZDQ).
* [Step 4: Test your integration](#h_01JD5CZBJBP7FYC7RY5NGJQF1V).

## Step 1: Retrieve InMobi Parameters

Before retrieving InMobi parameters, ensure that you have InMobi ad unit inventory that you want to mediate with DT. For more information about creating InMobi inventory, see the following InMobi articles:

* [Create](https://publisher.inmobi.com/signup) an InMobi account.

* [Create](https://support.inmobi.com/monetize/publisher-dashboard/inventory-tab/inventory-settings#add-inventory) an app in InMobi with the following considerations:

  * Verify the target audience to confirm if the application is specifically intended for children under 16.
  * Whether to authorize InMobi to access device location using *Location Permission*.
  * Consider using the Block Setting option selection if you are opting to enforce such restrictions.

* [Create](https://support.inmobi.com/monetize/publisher-dashboard/inventory-tab/inventory-settings#create-placements) a placements in InMobi for traditional waterfall mediation or SDK Bidding (Audience Bidding).

To set up inMobi as a DT mediated network, locate the following inMobi parameters:

| INMOBI PARAMETER | REQUIRED FOR              | DESCRIPTION                                                                                                                                    |
| ---------------- | ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `User Name`      | Waterfall                 | Your user name, as defined in the InMobi system, that you use to log in to your InMobi account.                                                |
| `ACCOUNT ID`     | * Waterfall
* SDK Bidding | The unique ID of your account in the InMobi system.                                                                                            |
| `API Key`        | Waterfall                 | A unique identifier for your InMobi account that allows the DT FairBid mediation platform to import performance data from your InMobi account. |
| `Placement ID`   | - Waterfall
- SDK Bidding | The unique Name/ID of a placement in the InMobi system.                                                                                        |

## Obtaining the API Key

1. Log in to your [InMobi account](https://login.microsoftonline.com/99b97e0e-6a13-4d32-b39a-1a2a28aa6ba8/oauth2/v2.0/authorize?redirect_uri=https%3A%2F%2Fpublisher.inmobi.com%2Fauth%2Fopenid%2Freturn\&client_id=f464a21e-8c1a-4717-aa29-86f89624762d\&scope=openid+profile\&response_type=id_token\&response_mode=form_post\&nonce=8f4vUXm084).

2) From the the drop-down at the top right-hand corner of your InMobi admin panel, go to **AccountSettings→API Key→ GenerateKey**.\
   A new API Key displays.

### Obtaining the Placement ID

## Step 2: Add InMobi as a Mediated Network

Before you add inMobi as a DT mediated network:

* Ensure that you have access to parameters you retrieved from inMobi in [Step1](#h_01J9W9C31TWB39TK3MG01T6EXE).
* [Add your App](https://developer.digitalturbine.com/hc/en-us/articles/14755391313308-Adding-an-App) using the DT Console.
* Ensure that you set up a [DT Placement](https://developer.digitalturbine.com/hc/en-us/articles/15005035293468-Setting-Up-Placements "Create a DT placement") for each inMobi ad type you want to mediate with DT.

To add InMobi as a mediated network:

1. Log in to the [DT Console](https://console.fyber.com/), and click**App Management**.

2) [Set up](https://developer.digitalturbine.com/hc/en-us/articles/15005016082972-Setting-Up-Mediated-Networks) **inMobi** or **InMobi bidder** as a mediated network using the parameters you retrieved in [Step1](#h_01J9W9C31TWB39TK3MG01T6EXE).

3. [Set up an ad network instance](https://developer.digitalturbine.com/hc/en-us/articles/15111147606428-Setting-Up-Instances) for each ad type you want to mediate with DT. Use the InMobi `Placement ID` you retrieved in [Step1](#h_01J9W9C31TWB39TK3MG01T6EXE) for the **Instance ID**.

## Step 3: Adding the InMobi SDK to your Integration

To integrate the inMobi SDK, follow the instructions on the [Supported Networks](https://developer.digitalturbine.com/hc/en-us/articles/360010077777-Supported-Networks) page.

## Step 4: Test Your Integration

Use the DT Test Suite to verify that you have set up your app properly for FairBid mediation. The DT Test Suite is available for Android, iOS, and Unity apps. For more information about using the DT Test Suite, see [Test Suite](https://developer.digitalturbine.com/hc/en-us/articles/16840582428316-Test-Suite "Using the DT Test Suite").
