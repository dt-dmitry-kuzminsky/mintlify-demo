This article is for publishers who want to use the DT FairBid SDK to load and display ads from Pangle via DT FairBid Mediation.

Pangle currently supports the following [ad types](https://developer.digitalturbine.com/hc/en-us/articles/360010915358-Ad-Types-and-Placement-ID):

* Banner
* Interstitial
* Rewarded

To integrate Pangle as a DT mediated network:

* [Step 1: Retrieve Pangle Parameters](#h_01J8PVB188B0R591QYA8GA8VZ9).
* [Step 2: Add Pangle as a DT mediated network](#h_01J8PVB189CN55GHV4E8PDT0ZA).
* [Step 3: Add the Pangle SDK to your integration](#01J8CVS1G0AC7G2TES6SKWE8DG).
* [Step 4: Test your integration](#01J8D022V8DN2D9FSKJXZA2VB1).

## Step 1: Retrieve Pangle Parameters

Before retrieving Pangle parameters, ensure you have the Pangle ad unit inventory you want to mediate with DT. For more information about creating Pangle inventory, see the following Pangle articles:

* [Create](https://www.pangleglobal.com/media/register "Create an Pangle account") a Pangle account.
* [Add](https://www.pangleglobal.com/integration/platform-configuration-for-mediation#1ud8r6953cf40 "Add your app to Pangle") your app to Pangle.
* [Create](https://www.pangleglobal.com/integration/platform-configuration-for-mediation#14lmore1ieyk0 "Create ad units in Pangle") ad units in Pangle for traditional waterfall mediation or SDK Bidding.

To set up Pangle as a mediated Network, locate the following Pangle parameters:

| PARAMETER         | REQUIREDFOR              | DESCRIPTION                                                        |
| ----------------- | ------------------------- | ------------------------------------------------------------------ |
| `Ad Placement ID` | * Waterfall
* SDK Bidding | Unique identifier of the Placement in the Pangle system.           |
| `API Key`         | Waterfall                 | Unique identifier to connect to your app data in the Pangle system |
| `App ID`          | - Waterfall
- SDK Bidding | Unique ID for your app in the Pangle system.                       |
| `Role ID`         | Waterfall                 | Unique identifier to connect to the Pangle Reporting API.          |
| `User ID`         | Waterfall                 | Unique identifier to connect to the Pangle Reporting API.          |



## Retrieving the App ID and Placement ID

1. On the [Pangle dashboard](https://www.pangleglobal.com/media/login), select **Applications**.\
   The`App ID` appears.

<Frame caption="">
  <img src="https://developer.digitalturbine.com/hc/article_attachments/4431642376849" />
</Frame>

2. Copy the`App ID`, and store the value in a text file. This value is required later in[Step2](#h_01J8PVB189CN55GHV4E8PDT0ZA)to add Pangle as a DT mediated network.
3. On the Pangle dashboard, click **Applications**and select the relevant App.\
   The`Placement ID` appears.
4. Copy the `Placement ID`, and store the value in a text file. This value is required later in[Step2](#h_01J8PVB189CN55GHV4E8PDT0ZA)to add Pangle as a DT mediated network.

Follow these steps to retrieve the `API Key`, and the `Role ID`fromthe Pangle dashboard:

1. On the[Pangle dashboard](https://www.pangleglobal.com/media/login), go to **Integrations > SDK & API > Pangle Reporting API 2.0**.\
   The parameters appear.

## Step 2: Add Pangle as a Mediated Network

Before you add Pangle as a mediated network in DT FairBid:

* Ensure that you have access to parameters you retrieved from Pangle in [Step1](#h_01J8PVB188B0R591QYA8GA8VZ9).
* [Add your App](https://developer.digitalturbine.com/hc/en-us/articles/14755391313308-Adding-an-App) using the DT Console.
* Ensure that you set up a [DT Placement](https://developer.digitalturbine.com/hc/en-us/articles/15005035293468-Setting-Up-Placements "Create a DT placement") for each Pangle ad type you want to mediate with DT.

To add Pangle as a DT mediated network:

1. Log in to the [DT Console](https://console.fyber.com/), and click**App Management**.

2) [Set up](https://developer.digitalturbine.com/hc/en-us/articles/15005016082972-Setting-Up-Mediated-Networks) **Pangle** or **Pangle Bidder** as a mediated network for your app using the parameters you retrieved in [Step1](#h_01J8PVB188B0R591QYA8GA8VZ9).

3. [Create an ad network instance](https://developer.digitalturbine.com/hc/en-us/articles/15111147606428-Setting-Up-Instances) for each placement you want to mediate with DT. Use the Pangle `Ad Placement ID`you retrieved in [Step1](#h_01J8PVB188B0R591QYA8GA8VZ9)as the **Instance ID**.

## Step 3: Adding the Pangle SDK to your Integration

To integrate the Pangle SDK, follow the instructions on the [Supported Networks page](https://developer.digitalturbine.com/hc/en-us/articles/360010077777-Supported-Networks).

#### Note

If you use ProGuard, follow the [Pangle instructions for SDK integration](https://www.pangleglobal.com/integration/integrate-pangle-sdk-for-android#as5ja83qobk0 "Pangle SDK integration instructions").

## Step 4: Test Your Integration

Use the DT Test Suite to verify that you have set up your app properly for FairBid mediation. The DT Test Suite is available for Android, iOS, and Unity apps. For more information about using the DT Test Suite, see [Test Suite](https://developer.digitalturbine.com/hc/en-us/articles/16840582428316-Test-Suite "Using the DT Test Suite").
